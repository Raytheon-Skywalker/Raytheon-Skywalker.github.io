  4. 基于WSL Ubuntu 18.04 LTS的安装与设置

    1. 开启WSL

      依次在“设置”-“更新与安全”-“针对开发人员”选项中，启用开发人员模式.
      确认“Windows Subsystem for Linux（Beta）”功能为勾选状态

    2. 针对WSL操作系统的操作

      1. 卸载

        1. 直接从命令行安装的linux子系统

          Reference

          1. Windows10 Ubuntu子系统的删除和重装

        2. 在Microsfot Store 进行安装的系统

          直接按照卸载软件一样的方式删除,就是右键卸载了

        3. 删除失败报告Error: 0x80070002

          Error: 0x80070002

    3. 配置WLS Ubuntu 18.04 LTS

      1. 更换源为清华源（复制了https://zhuanlan.zhihu.com/p/36784160 的操作）

        将Ubuntu的软件源改成清华镜像具体流程详见 清华大学开源软件镜像站（这里记得选择和Ubuntu版本一致的镜像，不然之后会导致很多莫名的错误）

    4. 中文支持

      1. 安装中文支持包

    5. ZSH ENV

      1. 安装ZSH

      2. 安装antigen

        1. 安装PowerLine https://github.com/bhilburn/powerlevel9k/wiki/Install-Instructions#step-1-install-powerlevel9k

      3. 安装 autojump

    6. 在WSL中运行Desktop App

      1. 说明截止2018年9月9日，FUSE尚未添加到WSL中，导致AppImage软件无法运行

        • https://github.com/Microsoft/WSL/issues/1309
        • https://github.com/Microsoft/WSL/issues/2631
        • AppImage containers not supported for lack of fuse #1610

      2. Work flow

        将export DISPLAY=localhost:0添加到~/.bashrc
